## 反编造守则
你输出的每一句话都必须基于输入原文中真实存在的信息。
规则：
1. 只引用原帖里明确出现的事实、动作、对象、结果。
2. 不要根据作者身份去脑补“他可能在说什么”。
3. 不确定就不写。
4. 不要用“他指出”“她强调”这类包装不确定内容。

你的任务是先做“信号抽取”，不是直接生成主题。

对每条 builder X 帖子，输出 1 条 signal，字段如下：
- `content_id`
- `source`
- `url`
- `topic_label`
  8-16 字中文标签，只写最核心的议题
- `core_claim`
  1 句中文，20-40 字，概括这条帖子到底在说什么
- `angle`
  从以下枚举选 1 个：`技术机制` / `具体案例` / `商业影响` / `个人体验` / `未来判断`
- `excerpt`
  1 句中文事实陈述，20-40 字，保留最能支撑判断的具体信息
- `spotlight_text`
  给“今日热议”降级展示用的中文句子，25-45 字
  只讲“发生了什么”，不要讲“为什么值得看”
  必须是具体事实句，优先包含动作、对象、结果
  不要写成“某人在讨论某问题”“某人提到某事”

规则：
- 每条帖子只允许 1 个 `topic_label`
- `topic_label` 尽量具体
- `excerpt` 只写最核心的新信息，不写评价句
- `spotlight_text` 必须是自然中文完整句
- `spotlight_text` 必须让读者一眼知道发生了什么

反例：
- `讨论对齐失败问题`
- `提到 agent 很重要`
- `聊了 GPT-5.5-Cyber`

正例：
- `未来几天将向关键网络防御者推出 GPT-5.5-Cyber 模型`
- `Box 正在招聘内部 Agent 工程岗位，并判断这会成为企业趋势`
- `Stripe 推出可供 AI 代理直接使用的数字钱包 Link`

如果原帖太弱，宁可保留最稳妥的具体事实，也不要写空泛句。

输出 JSON 结构：
{{
  "signals": [
    {{
      "content_id": "...",
      "source": "...",
      "url": "...",
      "topic_label": "...",
      "core_claim": "...",
      "angle": "...",
      "excerpt": "...",
      "spotlight_text": "..."
    }}
  ]
}}

严格输出 JSON，不要用 markdown 代码块包裹。

【输入数据】
Builder X 帖子（共 {n_posts} 条）：
{builder_posts}
